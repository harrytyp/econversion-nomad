elabFTW-NOMAD User Guide

Link elabFTW experiments and items to NOMAD entries — with automatic bidirectional cross-referencing.

Quick start: Create "elabFTW Settings" once with your API key. Then create "elabFTW Linked Entry" with an experiment ID. The normalizer links them automatically. Your key stays in your private settings entry.

Step 0: Prerequisites

  1. Log into elabFTW at https://elntest.ub.tum.de
  2. Go to My Account → API tokens and generate a token
  3. Log into NOMAD at https://researchmcp.duckdns.org/nomad-oasis/gui/

Step 1: One-time Setup — Create your Settings

  1. PUBLISH → Uploads → CREATE A NEW UPLOAD → CREATE FROM SCHEMA
  2. Select "elabFTW Settings"
  3. Name it exactly "elabFTW Settings" (the normalizer finds it by name)
  4. Fill in:
FieldValue
api_keyYour personal elabFTW token from My Account → API tokens
api_base_urlhttps://elntest.ub.tum.de/api/v2
  1. Save

Step 2: Upload Data and Link to elabFTW

Two ways to do this:

Method A: Upload files, then link (GUI)

  1. PUBLISH → Uploads → CREATE A NEW UPLOAD
  2. Upload your file(s) by drag-and-drop into the upload area
  3. Click CREATE FROM SCHEMA
  4. Select "elabFTW Linked Entry"
  5. Give it a descriptive name
Field (under config)Value
entity_typeexperiment (default) or item for database resources
sync_nowToggle to true
write_link_backKeep true
Field (under experiments)Value
elabftw_idThe elabFTW experiment ID (e.g. 2339) or item ID (e.g. 1354)
  1. Save

Method B: Create a new elabFTW experiment from NOMAD

  1. Same as above, but instead of filling in elabftw_id:
  2. Toggle create_elabftw_experiment to true
  3. Save — the normalizer creates a new experiment in elabFTW and links back

What the Normalizer Does

  • Reads your API key from your "elabFTW Settings" entry
  • Fetches the experiment/item title from elabFTW
  • Writes a nomad_url into the elabFTW experiment's metadata (not extra_fields — works with any category)
  • Clears any API key from the entry data (your key stays in your Settings entry)
|

Verify the Link

In elabFTW, open the linked experiment. Go to the Metadata section — you should see:

{
  "nomad_url": "https://researchmcp.duckdns.org/...",
  "nomad_synced": "2026-05-22T..."
}

Click the nomad_url to jump directly to your uploaded data in NOMAD.

Automated Uploads (API)

For scripts and machines uploading data automatically:

# 1. Get your NOMAD API token from Profile → API tokens
# 2. Upload your file(s) and create a linked entry in one call:

curl -X POST http://localhost:8000/nomad-oasis/api/v1/uploads \
  -H "Authorization: Bearer YOUR_NOMAD_TOKEN" \
  -F "upload_name=Sensor Data" \
  -F "file=@data.csv"

curl -X POST http://localhost:8000/nomad-oasis/api/v1/uploads/{upload_id}/archive \
  -H "Authorization: Bearer YOUR_NOMAD_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "entry_name": "Sensor Data May 22",
    "data": {
      "m_def": "three_way_sync.schema:ElabftwLinkedEntry",
      "config": {
        "entity_type": "experiment",
        "sync_now": true,
        "write_link_back": true
      },
      "experiments": [{"elabftw_id": "2339"}]
    }
  }'

The normalizer picks up your API key from your "elabFTW Settings" entry. No need to paste your key in the API call.

Using a Resource (database item) instead of an Experiment

Set entity_type: item in the entry config. The normalizer uses /api/v2/items/ instead of /api/v2/experiments/.

Creating a new elabFTW Experiment from Uploaded Data

If you want to create an elabFTW experiment automatically when you upload data to NOMAD, use the "elabFTW Machine Upload" schema instead. Set create_elabftw_experiment: true and the normalizer creates a new experiment in elabFTW with a link back to the NOMAD entry.

Security: Your API key is stored in your own private upload. Only you can see it. The normalizer reads it from your Settings entry and never stores it in linked entries. Other users cannot see your key even if they open your entries.
Permissions: The write-back of NOMAD URLs requires write permissions in elabFTW. If you get a 403 error, ask your elabFTW admin to grant you write access on the team.
