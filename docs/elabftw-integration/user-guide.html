elabFTW-NOMAD User Guide

Link elabFTW experiments and items to NOMAD entries — with automatic bidirectional cross-referencing.

Quick start: Create an "elabFTW Settings" entry once, then use "elabFTW Linked Entry" to link any experiment or item. Your API key stays in your private settings entry.

Step 1: One-time Setup — Create your Settings

  1. PUBLISH → Uploads → CREATE A NEW UPLOAD → CREATE FROM SCHEMA
  2. Select "elabFTW Settings"
  3. Name it exactly "elabFTW Settings" (the normalizer finds it by name)
  4. Fill in your elabFTW API token and base URL
  5. Save
FieldValue
api_keyYour personal elabFTW token from My Account → API tokens
api_base_urlhttps://elntest.ub.tum.de/api/v2

Step 2: Link an elabFTW Experiment/Item to NOMAD

  1. CREATE A NEW UPLOAD → CREATE FROM SCHEMA
  2. Select "elabFTW Linked Entry"
  3. Give it a descriptive name
Field (under config)Value
entity_typeexperiment (default) or item for database resources
sync_nowToggle to true
write_link_backKeep true
Field (under experiments)Value
elabftw_idID number (e.g. 2339 for experiment, 1354 for item)
  1. Save — the normalizer runs automatically

What the Normalizer Does

  • Reads your API key from your Settings entry
  • Fetches the experiment/item title from elabFTW
  • Writes a NOMAD URL back into elabFTW's extra fields
  • Clears the API key from the entry data

Verify the Link

In elabFTW, open the linked experiment/item. Under Extra fields you should see:

NOMAD URL: https://researchmcp.duckdns.org/...
NOMAD Synced: 2026-05-22T...

Machine Uploads

Use the "elabFTW Machine Upload" schema for automated uploads. Set create_elabftw_experiment to true, and the normalizer creates a new elabFTW experiment with a link back to the NOMAD entry.

Using a Resource (database item) instead of an Experiment

Set entity_type: item in the entry config. The normalizer uses /api/v2/items/ instead of /api/v2/experiments/.

Note: Your API key is stored in your own private upload. Only you can see it. The normalizer reads it from your Settings entry and clears it from linked entries after processing.

Troubleshooting

  • "No API key" — create an ElabftwSettings entry first
  • Link not written — check elabFTW API key has write permissions
  • Wrong entity type — set entity_type to "item" for database resources
  • Schema not visible — log in and check CREATE FROM SCHEMA lists "elabFTW Settings" and "elabFTW Linked Entry"
